# Testing Tools - Quick Reference

## Overview

Your print system now has comprehensive error tracking and diagnostic tools. Here's what's available:

---

## ğŸ”§ Diagnostic Tools

### 1. **Diagnostic Report Generator**
- **File**: `src/print_diagnostics.py`
- **Quick Run**: Double-click `run_diagnostics.bat`
- **What it does**:
  - Checks system information
  - Lists all available printers
  - Verifies required modules
  - Tests preset configuration
  - Checks database status
- **Output**: `print_diagnostic_report.txt`
- **When to use**: **FIRST THING** when testing on a new machine

### 2. **Error Logger**
- **File**: `src/error_logger.py`
- **Automatic**: Runs in background during app usage
- **What it does**:
  - Logs all errors with timestamps
  - Captures full stack traces
  - Records context (order numbers, printer names, etc.)
- **Output**: `print_errors.log`
- **When to use**: Automatically captures errors as they happen

### 3. **Log Viewer** (Built into App)
- **File**: `src/log_viewer.py`
- **Access**: Will be added to app menu/toolbar
- **What it does**:
  - Shows recent errors in a nice UI
  - Copy button to share logs
  - Refresh to see new errors
  - Clear log button
- **When to use**: To quickly check what errors occurred

## ğŸ¯ Quick Troubleshooting

### "No printers found"
```
1. Check print spooler service is running
2. Run: run_diagnostics.bat
3. Send me the report
```

### "Module not found"
```
1. Run: pip install pywin32 PyPDF2 python-docx
2. Try again
3. If still fails, send error message
```

### "Timeout errors"
```
This is normal for slow print servers!
- Check print_errors.log
- Note which printer is slow
- Tell me the timeout duration you need
```

### "Preset editor blank"
```
1. Click on preset name in left list
2. Take screenshot
3. Send: print_errors.log + screenshot
```
